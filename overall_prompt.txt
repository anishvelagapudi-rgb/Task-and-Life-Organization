AUTHENTICATION REQUIREMENTS
====================================================

This app is single-user internally but must still have secure authentication.

Implement:
- Google OAuth 2.0
- session-based auth
- protected routes

DO NOT implement:
- registration
- onboarding
- teams
- organizations
- RBAC
- collaboration

Authentication logic:

1. User logs in via Google OAuth
2. Backend verifies email
3. Backend checks if email matches allowed owner email
4. If valid:
   - create session
   - grant access
5. Otherwise deny access

This is a private system.
